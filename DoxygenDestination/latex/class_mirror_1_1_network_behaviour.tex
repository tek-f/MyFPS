\doxysubsection{Detailed Description}
Base class which should be inherited by scripts which contain networking functionality. 

This is a Mono\+Behaviour class so scripts which need to use the networking feature should inherit this class instead of Mono\+Behaviour. It allows you to invoke networked actions, receive various callbacks, and automatically synchronize state from server-\/to-\/client.

The \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} component requires a \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} on the game object. There can be multiple Network\+Behaviours on a single game object. For an object with sub-\/components in a hierarchy, the \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} must be on the root object, and \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} scripts must also be on the root object.

Some of the built-\/in components of the networking system are derived from \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}}, including Network\+Transport, \mbox{\hyperlink{class_mirror_1_1_network_animator}{Network\+Animator}} and \mbox{\hyperlink{class_mirror_1_1_network_proximity_checker}{Network\+Proximity\+Checker}}.

Virtual function to override to send custom serialization data. The corresponding function to send serialization data is \mbox{\hyperlink{class_mirror_1_1_network_behaviour_a26853c614ebc9c116a7d4f045d723be1}{On\+Deserialize()}}. 

The initial\+State flag is useful to differentiate between the first time an object is serialized and when incremental updates can be sent. The first time an object is sent to a client, it must include a full state snapshot, but subsequent updates can save on bandwidth by including only incremental changes. Note that Sync\+Var hook functions are not called when initial\+State is true, only for incremental updates.

If a class has Sync\+Vars, then an implementation of this function and \mbox{\hyperlink{class_mirror_1_1_network_behaviour_a26853c614ebc9c116a7d4f045d723be1}{On\+Deserialize()}} are added automatically to the class. So a class that has Sync\+Vars cannot also have custom serialization functions.

The On\+Serialize function should return true to indicate that an update should be sent. If it returns true, then the dirty bits for that script are set to zero, if it returns false then the dirty bits are not changed. This allows multiple changes to a script to be accumulated over time and sent when the system is ready, instead of every frame.
