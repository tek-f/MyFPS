\doxysubsection{Detailed Description}
The \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} identifies objects across the network, between server and clients. Its primary data is a Network\+Instance\+Id which is allocated by the server and then set on clients. This is used in network communications to be able to lookup game objects on different machines. 

The \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} is used to synchronize information in the object with the network. Only the server should create instances of objects which have \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} as otherwise they will not be properly connected to the system. 

For complex objects with a hierarchy of subcomponents, the \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} must be on the root of the hierarchy. It is not supported to have multiple \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} components on subcomponents of a hierarchy. 

\mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} scripts require a \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} on the game object to be able to function. 

The \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} manages the dirty state of the Network\+Behaviours of the object. When it discovers that Network\+Behaviours are dirty, it causes an update packet to be created and sent to clients. 

The flow for serialization updates managed by the \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} is\+: 
\begin{DoxyItemize}
\item Each \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} has a dirty mask. This mask is available inside On\+Serialize as sync\+Var\+Dirty\+Bits  
\item Each Sync\+Var in a \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} script is assigned a bit in the dirty mask.  
\item Changing the value of Sync\+Vars causes the bit for that Sync\+Var to be set in the dirty mask  
\item Alternatively, calling Set\+Dirty\+Bit() writes directly to the dirty mask  
\item \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} objects are checked on the server as part of it\textquotesingle{}s update loop  
\item If any Network\+Behaviours on a \mbox{\hyperlink{class_mirror_1_1_network_identity}{Network\+Identity}} are dirty, then an Update\+Vars packet is created for that object  
\item The Update\+Vars packet is populated by calling On\+Serialize on each \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} on the object  
\item Network\+Behaviours that are N\+OT dirty write a zero to the packet for their dirty bits  
\item Network\+Behaviours that are dirty write their dirty mask, then the values for the Sync\+Vars that have changed  
\item If On\+Serialize returns true for a \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}}, the dirty mask is reset for that \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}}, so it will not send again until its value changes.  
\item The Update\+Vars packet is sent to ready clients that are observing the object  
\end{DoxyItemize}

On the client\+: 
\begin{DoxyItemize}
\item an Update\+Vars packet is received for an object  
\item The On\+Deserialize function is called for each \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} script on the object  
\item Each \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} script on the object reads a dirty mask.  
\item If the dirty mask for a \mbox{\hyperlink{class_mirror_1_1_network_behaviour}{Network\+Behaviour}} is zero, the On\+Deserialize functions returns without reading any more  
\item If the dirty mask is non-\/zero value, then the On\+Deserialize function reads the values for the Sync\+Vars that correspond to the dirty bits that are set  
\item If there are Sync\+Var hook functions, those are invoked with the value read from the stream.  
\end{DoxyItemize}

\begin{DoxyItemize}
\item Ideally we would have a serialized \textquotesingle{}Guid m\+\_\+\+Asset\+Id\textquotesingle{} but Unity can\textquotesingle{}t serialize it because Guid\textquotesingle{}s internal bytes are private  
\item U\+N\+ET used \textquotesingle{}Network\+Hash128\textquotesingle{} originally, with byte0, ..., byte16 which works, but it just unnecessary extra code  
\item Using just the Guid string would work, but it\textquotesingle{}s 32 chars long and would then be sent over the network as 64 instead of 16 bytes  
\end{DoxyItemize}The solution is to serialize the string internally here and then use the real \textquotesingle{}Guid\textquotesingle{} type for everything else via .asset\+Id \mbox{\Hypertarget{class_mirror_1_1_network_identity_a2c3c6ec4b03d4d8212a478fe0877f638}\label{class_mirror_1_1_network_identity_a2c3c6ec4b03d4d8212a478fe0877f638}}
